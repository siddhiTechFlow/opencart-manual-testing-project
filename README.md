# OpenCart Manual Testing Project

## Project Overview

This repository contains a manual testing project conducted on the OpenCart demo e-commerce application. The project demonstrates the complete manual testing workflow including requirement understanding, test case design, test execution, compatibility testing, exploratory testing, and defect reporting.

Test cases and testing activities were managed using Bugasura and exported as documentation for this repository.

---

## Application Under Test

OpenCart Demo Website
Application Type: E-commerce Web Application
Testing Type: Manual Testing

---

## Objective

The objective of this project is to validate the functionality and usability of the OpenCart e-commerce application by designing and executing structured test cases. The testing focuses on ensuring that important user workflows such as registration, login, product search, and product display work correctly according to the expected system behavior.

Different testing techniques were applied to identify defects, validate system behavior, and ensure that the application performs reliably under various conditions.

---

## Scope of Testing

The testing activities focused on validating the core features of the OpenCart application.

Modules covered in this testing project include:

* User Registration
* User Login
* Product Search
* Product Display

## Test Case Design

Test cases were designed and executed using Bugasura. Each test case includes structured fields such as:

* Test Scenario
* Test Type
* Severity
* Priority
* Feature and Sub-Feature
* Test Conditions
* Test Steps
* Acceptance Criteria

These structured test cases help ensure proper validation of application functionality under different test scenarios.
